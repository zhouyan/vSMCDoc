Detailed Description

template<typename ID, typename ResultType = std::uint32_t, std::size_t K = 4>
class vsmc::SeedGenerator< ID, ResultType, K >

Seed generator.

Template Parameters
IDDifferent ID type create indepdent SeedGenerator instances
ResultTypeThe unsigned integer type of the seeds
KThe length of the key. A key is a std::array<ResultType, K> type object

The sequence of seeds are belongs to the equivalent class \(s \mod D \equiv R\) where \(D > 0\), \(R \ge 0\). The defaults are \(1\) and \(0\) respectively. Each time get() is called, a new seed is returned.

The sequence of keys are of the form \(\{s_1,s_2,\dots,s_{K - 1},s\}\). The first \(K - 1\) elements forms a counter. Each time get_key() is called, the counter is incremented and a key that combine the counter and the current seed is returned.

The method operator()(RNGType &rng) is equivalent to rng.seed(get_key()) if RNGType::key_type exists and it is the same type as key_type of this class. Otherwise, it is equivalent to rng.seed(get()). This method is thread-safe but slower than rng.seed(get()) or rng.seed(get_key()), which are not thread-safe.

The only thread-safe mutable method is operator(). Seeding RNGs from multiple threads at the same time using this method is possible. Any other mutable methods are not thread-safe. In particular, if this singleton is going to be used in a multi-threaded environment, make sure instance() is called at least once in a single threaded environment.
